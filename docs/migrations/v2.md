# html2rss-web v2 Migration Guide

This guide covers the key behavioral and operational changes to move from pre-v2 releases to v2.

## 1) Runtime Port Default

- New default app port is `4000`.
- If your deployment assumed `3000`, set `PORT=3000` explicitly or update your ingress/proxy.

## 2) Health Endpoints

v2 exposes three health routes:

- `GET /api/v1/health` (authenticated)
- `GET /api/v1/health/ready` (unauthenticated)
- `GET /api/v1/health/live` (unauthenticated)

Update orchestrator probes to use `ready/live` where auth headers are not available.

## 3) Error Response Hardening

- Unexpected internal failures now return a generic error message to API clients.
- Custom `HttpError` responses remain user-facing as before.

If client logic depended on raw exception strings, switch to `error.code`.

## 4) Auto Source Feature Flag

Feature behavior is explicit by environment policy:

- Development: enabled unless `AUTO_SOURCE_ENABLED=false`
- Non-development: enabled only when `AUTO_SOURCE_ENABLED=true`

For smoke/CI flows, set `SMOKE_AUTO_SOURCE_ENABLED` to match intended runtime mode.

## 5) Auth/Error Semantics on Feed Endpoints

Feed endpoint ordering is now consistent:

- `401 UNAUTHORIZED` for missing/invalid credentials
- `403 FORBIDDEN` for authenticated requests that violate policy (e.g. auto source disabled, URL not allowed)

## 6) Deployment Checklist

1. Set `HTML2RSS_SECRET_KEY` to a strong value.
2. Confirm `PORT` and external routing match your environment.
3. Set `AUTO_SOURCE_ENABLED` intentionally for production.
4. Verify probes hit:
   - `/api/v1/health/ready`
   - `/api/v1/health/live`
5. Run smoke checks in both feature modes when applicable.
